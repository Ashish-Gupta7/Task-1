1. clone - cloning a repository on our local machine(laptop/computer...)
git clone <-some link->
2. status - displays the state of the code.
git status
3. cd(change directory) command.
4. terminal ko clear krne ke liye clear command ka use krte h.
5. terminal me sabhi files ko dekhne ke liye ls(list files) command ka use krte h.
6. hidden files ko dekhne ke liye ls -a command ka use krte h.
7. U ~ untracked files aisi file jise git track nhi kr rha h. jb bhi hm new file banate h to vo file untracked file hoti h.
8. M ~ modified files aisi file jise modify kiya gya h.
9. jb bhi hm modified/untracked file ko add kr dete h to vo staged(file is ready to be committed) ki form me chli jati h.
10. aur jaise hi hm staged ki stage ko commit kr dete h to vo phir se unmodified ki stage me chla jata h. mtlb ab file modify h ya new h iski jankari git ko mil gyi hoti h isliye commit krna importent hota h.
11. Add - adds new or modified files in your working directory to the Git staging area.
git add <-file name->
12. commit - it is the record of change(modify).
git commit -m"some message"
13. Push Command -
push - upload local repo(jo files computer system me h) content to remote repo(jo files github pr h).
git push origin main
